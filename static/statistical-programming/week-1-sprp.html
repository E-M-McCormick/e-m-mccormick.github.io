Programmatic Statistics

What is “Programmatic Statistics”?

  • An approach to data management, manipulation, analysis, and reporting that incorporates scientific programming to minimize repeated effort, improve reproducibility and replicability, and modularize common tasks.


  • Large range of flexibility: could use to organize individual data files, recode and manipulate data frames, run and report statistical analyses, visualize results, or even write entire manuscripts.

What is “Programmatic Statistics”?

  • In this class, we will focus on two languages: R & Python
    • freely-available
    • functional and/or object-oriented
    • large and well-documented user-driven environments

Examples

R

fit <- lm(y ~ x, data = data)
summary(fit)

Python

mod = sm.OLS(y, X).fit()
mod.summary()

Broad Generalizations

  • R
    • broad statistical modeling capabilities
    • good libraries for data management and visualization
    • function-based (more intuitive), easy parallel computing
  • Python
    • general coding & all-purpose programming
    • machine learning applications & big data
    • object-oriented, stronger typing & classes

Regression Models

Regression Models

The core statistical model we will use is the regression model.

Y_i = \beta_0 + \beta_1 X_{i} + \varepsilon_i \qquad \varepsilon_i \sim \mathcal{N}(0, \sigma^2) The outcome (Y_i) for individual i is modeled as a function of a conditional mean (\beta_0) where all covariates (X_i) are 0, some effect (\beta_1) of a one-unit difference in X_i, and a random error term (\varepsilon_i). The random error term is normally-distributed (\mathcal{N}) with a mean of zero and a variance estimated by \sigma^2.

Inference vs. Prediction

Inference vs. Prediction

  • Inference (explanatory models): The goal is to map (hopefully) causal relations between constructs of interest.

    • Interested in model parameters, uncertainty, “significance”, and substantive conclusions

  • Prediction (predictive models): The goal is to successfully predict subsequent observations from a trained model.

    • Performance on new observations is key, model parameters are of less (and often no) interest.

Inferential Models

  • Preference for simpler, but interpretable, models

  • Reliance on estimates of uncertainty and assumptions (e.g., standard errors, p-values) to guide conclusions

  • The goal is to find the “correct” (or at last “correct-ish”) model

Predictive Models

  • Preference tends to be for more complex models, at the cost of interpretability

  • Reliance on measures of error (e.g., MSE, mis-classification) to guide model selection

  • Goal is to find a model that generalizes to new observations, sometimes even when implausible or explicitly incorrect

Versus or Both?

  • In reality, most applications need some balance of both inference and prediction

    • An inferential model that doesn’t make good predictions cannot be “correct” in some sense

    • Predictive models are incredibly prone to overfit

  • Need to be clear about the goal of an analysis to balance
